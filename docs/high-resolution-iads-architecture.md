# 고해상도 IADS 목표 아키텍처 경계

> 본 문서의 좌표·성능·확률은 공개자료 기반 **정책연구용 개념값**이며 실제 작전자료가 아니다.

이 문서는 고해상도 구현의 **후속 목표 구조**를 고정한다. 2026-07-19 후속 변경으로 위협 종류·아키텍처·C2 생존상태 책임 Resolver, scope별 사수선정, 개념 3D 축선 PIP/교전봉투, 발사대별 탄약·900초 재장전, 실제 발사 기반 중복교전·BDA는 고해상도 경로에 구현되었다. 아래의 정본 이벤트, 도메인별 RNG, 실시간 센서/항적 상태, 정밀 PSSEK 테이블은 아직 목표 경계다.

## 1. 정본 이벤트 스키마

모든 상태 변경은 `eventId`, `eventType`, `simTime`, `entityId`, `causationId`, `correlationId`, `payload`, `schemaVersion`을 가진 이벤트로 표현한다. 위협 운동, 센서 스캔, 항적 갱신, C2 메시지, 교전·BDA, 재장전을 서로 다른 이벤트 유형으로 둔다. 엔진은 이벤트 저장소와 뷰 프로젝션을 분리해 재생·감사가 가능해야 한다.

## 2. 도메인별 RNG

도착, 위협 운동, 센서 탐지, 항적 유지, C2 처리, 사수 선정, 명중/BDA, 재장전에 서로 다른 파생 시드를 배정한다. 특정 모델 상세도를 켜거나 끄어도 무관한 도메인의 난수 소비 순서가 바뀌지 않아야 한다. paired MC는 도메인 스트림별 CRN을 선택적으로 공유한다.

## 3. 위협 운동학·센서·항적

- 위협은 3차원 위치·속도·가속도·경로 세그먼트를 가지며, 교전창은 고정 `dwellSec`가 아닌 기하·시간으로 판정한다.
- 센서는 전원/정비/상태, 주사주기, 보임, 탐지·측정 오차, 데이터 지연을 가진다.
- 항적은 탐지와 분리된 엔티티로 표현하고 `quality`, `freshness`, 공분산, 소스 목록, 소실/재획득 상태를 유지한다.

## 4. PIP/PSSEK

현재 엔진은 개념 축선 위치·고도, 요격체 속도, `Rmin/Rmax/Hmin/Hmax`, 명령 지연으로 PIP 실현 가능성과 발사 시각을 계산한다. 후속 구조에서는 `PIP`(예측 요격점)와 `PSSEK`(사수–센서–교전 키)를 별도 서비스로 분리하고, 항적 공분산·조우각·정밀 성능 테이블을 추가한다.

## 5. C2 Resolver

현재 Resolver는 위협 종류, As-Is/To-Be 아키텍처, C2 생존상태와 force owner를 입력받아 KAMDOC/MCRC/ICC/IAOC/지역방공/USFK 독립 축을 결정한다. 후속에서 ROE, AOR, track-release authority, 동적 delegation state를 명시적 정책 객체로 분리한다. 한국군 KAMDOC/MCRC와 USFK THAAD/Patriot 권한은 현재도 자동 통합하지 않는다.

## 6. 호환 어댑터 폐기 조건

동일 배치에 대해 이벤트 스키마, 도메인 RNG, 위협 운동, 센서/항적, PIP, C2 Resolver, 차량별 재장전이 모두 결합되고 보존법칙·결정론·paired MC·V&V 게이트를 통과한 후 `phase1-axis-queue` 어댑터를 폐기한다.
