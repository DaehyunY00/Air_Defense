# K-JAMDS C2 병목 분석 시뮬레이터 — V&V 보고서 (Phase 5)

> **디스클레이머**: 본 문서와 시뮬레이터의 모든 수치·좌표·확률은 공개자료(오픈소스) 기반의 **정책연구용 개념값**이며, 실제 작전자료가 아님. 모든 좌표는 도시 수준 개념좌표임. KP-SAM(신궁)·천마(K-31)는 탄도탄 요격 불가로 모델링하며, KAMDOC↔THAAD 연동은 모델링하지 않음. 본 시뮬레이터는 절대값 예측 도구가 아니라 **As-Is/To-Be 상대비교** 도구다.

## 3. Validation — 모델 타당성

### 3.1 극한값 검증 (계획서 4절 V&V)

- **위협 0 / 강도 0**: 생성·병목·부하 전부 0, NaN 없음.
- **초과포화 (강도 3.0)**: 이벤트 루프 정상 종료(무한루프 없음), 관측 ρ≤1(M/M/c/K 물리 상한), 드롭·누수 발생, 병목 도출 — 대기열 이론의 예측과 정성적 일치.

### 3.2 실증 사례 face validity — 2022.12.26 무인기 침투 재현 (SC1)

As-Is 모드에서 모델이 재현하는 구조가 공개된 사건 전개와 정성적으로 일치한다:

| 실제 사건 (공개자료) | 모델 관측 |
|---|---|
| 1군단 국지방공레이더 최초 포착, 항적이 국가방공체계에 미통합 | 육군 센서(LLR-1C) 탐지 → AOC-1C 보유 항적이 MCRC로 음성(180초) 협조로만 전달 — 해당 voice 링크가 통신병목으로 도출 |
| 공작사·항작사·군단·수방사 개별 작전(중복·책임공백) | 중복교전 히트맵: 서부·수도권 축선에서 As-Is 위험>0, To-Be(JAMDC2 융합)에서 0으로 해소 |
| 격추 실패 (헬기·전투기 약 20대 투입) | 소형 무인기 요격확률 Triangular(0.1,0.3,0.5) — 표본 실행에서 무인기 격추율이 타 위협 대비 현저히 낮음 |
| 5시간 이상 체류 | dwellSec=900(최장) + 저탐지(detectFactor 0.4) 재획득 반복 |

### 3.3 민감도 분석 (±20% 스윕, 계획서 4절)

SC3 포화 시나리오(As-Is, 강도 1.5)의 누수율 스윙: **노드 처리시간 ~28%p ≫ 위협 강도 ~18%p ≫ 요격확률 ~9%p ≫ 통신지연 ~1.5%p ≈ 탐지확률 ~0.6%p**.
- 포화 구간의 지배 인자는 처리용량 — "병목은 처리용량"이라는 진단의 정량적 근거.
- 대조: 탐지 제약 시나리오(SC1)에서는 탐지확률↑→누수↓ 단조성이 성립 — 민감도 구조 자체가 시나리오의 함수임을 확인(모델이 사전 결론을 내장하지 않음).

### 3.4 임계 전환점 — As-Is→To-Be 효과 정량화 (계획서 Recommendations 6, 핵심 산출물)

SC3 복합 섞어쏘기 포화, 강도 0.5×~3.0× 스윕(11점×2모드×30복제, seed=12345, 결정론적):

| 구간 | As-Is C2 최대 ρ | As-Is 누수율 | To-Be 누수율 | 개선폭(gap) |
|---|---|---|---|---|
| ×0.5 (저강도) | 0.33 | 44.4% | 36.4% | 8.0%p |
| ×1.5 (임계 직전) | 0.86 | 54.8% | 41.6% | 13.2%p |
| **×1.75 (ρ≥0.9 돌파)** | **0.93** | 60.4% | 42.6% | 17.8%p |
| ×2.5 (최대 격차) | 0.98 | 70.1% | 49.3% | **20.8%p** |
| ×3.0 | 0.98 | 72.6% | 56.2% | 16.4%p |

- **임계 이전 평균 개선폭 10.3%p → 임계 이후 18.9%p** (약 1.8배).
- 정책 함의: To-Be(Track Fusion·자동 WTA·데이터링크 30초→2초)의 가치는 평시·저강도가 아니라 **결심권자 이용률이 임계(ρ=0.9)를 넘는 포화 구간에서 비선형적으로 발현**된다. 이것이 K-JAMDS 투자정당화의 정량 논거다.
- 대조군: SC1(경계 침투, 저부하)은 전 스윕 구간 ρ<0.9(전환점 부재) — 전환점 역시 시나리오에서 도출되는 값이지 고정 속성이 아니다. (Phase 6에서 시나리오가 KJADS 3대 문제 상황으로 재편됨 — SC1 교전중복·SC2 무인기 burst·SC3 섞어쏘기)
- UI: [Monte Carlo] 탭 "임계 전환점" 카드에서 대화형 재현 가능(동일 seed → 동일 곡선).

## 4. 한계 (계획서 Caveats 승계 + 구현 고유)

2. **모델 단순화**: 3·4·5단계(식별·위협평가·WTA)를 단일 C2 서비스시간으로 통합, 지수 서비스시간 가정(M/M/c), 교전은 요격확률 단발 판정(비행시간·기하 미모델링), 위협 궤적은 시각화용 선형보간.
3. **합참지 제104호 원문 미확보**: K-JAMDS 세부 약어(JAMDC2·KIECS 등)는 사용자 제공 개념으로 처리(계획서 Caveats).
4. **trace 절삭**: 재생용 위협 추적은 300건, 노드 시계열은 20,000샘플 상한(초과 시 truncated 플래그 명시) — **통계 결과는 절삭과 무관하게 전체 모집단 기준**(회귀로 검증).
5. **Mulberry32**: 32비트 주기 — 본 규모(복제 수백 회)에는 충분하나 초대규모 배치는 장주기 생성기 검토 필요.
6. **비용교환비(Phase D)**: 요격탄·위협 단가가 타 전역 공개수치 기반 개념값(WPN/THR-*-COST-01) — 한반도 획득단가 보정 전에는 절대값이 아닌 모드 간 상대비교로만 해석. 교전 시도 1회=1발 소모 개념(일제사·재고 미모델링).
7. **PRA(일제사 전멸확률) 미구현**: burst 일제사 전량 요격 확률(MoFE 후보)은 파도(wave) 단위 추적이 필요해 이번 정밀화에서 제외 — 향후 MC 러너에 wave 집계 추가로 구현 가능.
8. **동적 권한위임의 단순화(Phase B-2)**: 위임은 승인 홉 생략으로만 모델링(위임 후 하위 제대의 추가 처리부하·교전규칙 차이 미모델링). 임계(C2-DELEG-THRESH-01)는 개념값.

### 4-1. ① 탐지 융합(센서 Pd, `feat/sensor-pd-fusion`)의 잔여 한계

탐지 확률을 [센서 Pd × 위협난이도 → 모드별 융합]으로 배선했으나(SEN-FUSION-01), 이번 범위로도 **해결되지 않는 것**을 정직하게 기재한다:

- **탐지율은 여전히 축퇴에 가깝다(N-포화)**: 스캔 재시도 상한이 체공(dwellSec)이라 시행횟수 N=dwell/10이 커, 체공 위협의 누적 탐지율은 As-Is·To-Be 모두 ~100%로 포화한다. 융합의 효과는 탐지 **"율"이 아니라 "시점"** 과 종료 절단 꼬리에서만 관측된다(정량: `tests/detect.test.js` 축3 — 수도권 무인기 중위 30s→10s, 단일센서 중부 무인기 30s=30s 불변).
- **항적 연속성(track continuity) 미구현**: 한 번 `detected=true`가 되면 영구 유지된다. 2022.12.26의 실제 문제("탐지 후 반복 소실")는 여전히 모사되지 않는다. → 옵션 C 과제.
- **센서 독립성 가정**: 융합식 `1−∏(1−pᵢ)`는 센서 간 독립을 가정한다. 실제로는 저RCS·기상·지형에 대해 탐지 실패가 상관되어 **To-Be 융합 효과가 과대추정될 수 있음.**
- **종료 절단(censoring) 미보정**: `spawnT + dwellSec > endTimeSec`인 위협이 분모에 남아 탐지율을 ~1%p 끌어내린다(시뮬레이션 길이가 지표를 바꿈). → 별도 이슈.
- **주야 구분 미구현**: ADC2A-W 육안관측 Pd는 주간 0.5 / 야간 0.2로 문서화(SEN-JASP-PD-01)되어 있으나 단일 혼합값 0.35로 처리했다.
- **거리·RCS 미반영**: 센서 `rangeKm`은 데이터에 있으나 탐지 확률 계산에 쓰이지 않는다(커버리지는 축선 이진 판정). per-scan Pd는 표적·거리 무관 고정값이다.

### 4-2. ② 추적생성·보고링크(`feat/stage2-track-overhaul`)의 잔여 한계

②단계를 [report/coord/command 지표 분리 · 음성 지연 용도별 분포화 · To-Be 센서→JAMDC2 직결 · As-Is 중복항적 팬아웃]으로 개편했으나, 이번 범위로도 **해결되지 않는 것**을 정직하게 기재한다:

- **중복교전(duplicate engagement) 미구현 → [GitHub 이슈 #5](https://github.com/DaehyunY00/Air_Defense/issues/5)**: Phase 4 팬아웃은 "C2 부하 중복"만 모사한다. **동일 표적에 두 통제계통이 각각 사격하는 중복교전은 여전히 모사되지 않는다.** 이는 SC1(문제상황 1)의 본질이며, 구현하면 요격탄 이중 소모 → **비용교환비(MoFE)가 As-Is에서 악화**되는, 지금은 보이지 않는 효과가 나타난다(현재 MoFE는 As-Is 중복교전 비용을 **과소평가**). → 별개 과제.
- **항적 연속성(track continuity) 미구현**: `detected=true`가 영구 유지되어 "탐지 후 반복 소실"이 미모사(옵션 C). 재획득 dup(`config.dupReacquire`)은 기본 off로 예약만 되어 있고, 켜려면 **배수가 아닌 이벤트로 구현**해야 함(근거 없는 `1/detectFactor` 배수는 스캔 재시도와 이중계상 위험 — 실측: 켜면 SC3 killed 78% 붕괴). → 범위 밖.
- **음성 지연 분포의 독립성 가정**: `_linkDelay`의 삼각분포 샘플링은 홉·항적 간 **독립**을 가정한다. 실제로는 통신 폭주 시 지연이 **상관**되어 As-Is 병목이 과소평가될 수 있음.
- **`no_report_path`는 구조적으로 발화 불가능한 死 코드**: 커버 센서가 있으면 report 링크는 항상 존재하므로 이 실패코드는 발화하지 않는다. taxonomy에는 남기되(문서화 목적) 관측되지 않음을 명시. `not_detected`는 ①단계(센서 Pd) 소관.
- **Phase 5 死링크 재측정(전 시나리오·모드·강도·seed 누계)**:
  - `ADC2A-W→AOC-1C`(음성 report): 종전 0 → **부활 316건**. 팬아웃 후 cruise@west·seoul는 LLR(무인기/저속기/헬기만 커버)이 못 봐 AOC-1C로 가는 최속 링크가 ADC2A-W가 되어 발화한다.
  - `LLR-CD→JAOC-CD`: 종전 0 → **부활 1,704건**(팬아웃). JAOC-CD 死노드 탈출.
  - `LLR-1C→JAMDC2`·`LLR-CD→JAMDC2`(Phase 3 직결): **0건 — 새 死 링크**. To-Be 직결 tiebreak(센서 id 사전순)에서 커버리지 상위집합인 `ADC2A-W`가 항상 이겨, 육군 항적은 JAMDC2에 직행하되(`ADC2A-W→JAMDC2` 5,102건) 이 두 링크는 기록되지 않는다. 기능(육군 plot→JAMDC2 직행)은 충족되므로 조용히 삭제하지 않고 **중복 死 링크로 명시**한다(단일 항적 모델의 표현 한계 — 실제로는 다센서 plot이 동시에 융합 입력이 됨).
- **팬아웃은 As-Is만 무겁게 만든다(방향성 주의)**: 중복항적은 정의상 As-Is에만 부과되고 To-Be는 Track Fusion이 흡수한다. 이는 결론에 유리한 방향이나, 팬아웃 대상(커버 C2 집합)은 **센서 커버리지 데이터에서 도출**되어 임의 배수가 아니다(등급 B). 근거 없는 값은 얹지 않았다.

### 4-3. ③④⑤ 식별·위협평가·WTA / ⑥⑦ 승인처리(`feat/stage345-c2-metrics`)의 잔여 한계

③④⑤·⑥⑦ 지표를 [C2 부하 track/approval 분리 · Wq MoP 노출 · ρ 해석 주의 · C2 K 근거 명시(`[ENV-DES-C2K-01]`)]로 개편했으나, 이번 범위로도 **해결되지 않는 것**을 정직하게 기재한다:

- **③④⑤ 세부 미분리(단일 지수분포 서비스)**: 식별(③)·위협평가(④)·WTA(⑤)는 `_startService`의 지수분포 서비스 **1회**로 뭉뚱그려져 개별 관찰이 불가능하다. 카드의 ③④⑤ 지표는 세 단계 합산 C2 처리부하(track)만 측정한다. 세 단계를 별도 서비스 스테이지로 쪼개는 것은 이번 범위 밖. → 별개 과제.
- **⑤ WTA 실행 위치가 교리 순서(F2T2EA)와 다름**: `_doEngage`(WTA)는 ③④⑤가 아니라 ⑥⑦ 결심(`_decision`) **이후** ⑧ 단계에서 실행된다(엔진 실행 순서 ≠ 교리 순서). 카드 `fn`·툴팁·`docs/분석가_가이드.md`에 명시했으나, 카드 배열 자체는 교리 순서를 유지하므로 독자가 순서 차이를 인지해야 한다.
- **C2 대기실 K 근거 부재(등급 C)**: C2 노드 capacity(K=15/30/40/60)는 서버 수 대비 배수로 임의 설정된 값으로 공개근거가 없다(`[ENV-DES-C2K-01]`에서 등급 C·민감도 스윕 대상으로 명시). 버스트 시나리오에서 결과를 결정할 수 있는 파라미터이나 근거 등급이 가장 낮다. As-Is 버스트에서 K가 실제로 구속함을 실측(SC2 x2.5 AOC-1C K=15 드롭 7.1건, 임계 K≈30)했으나, 드롭 제거가 outcome(격추율·누수율)을 바꾸지 않아(무인기 저Pk 구속) K 튜닝은 결론을 흐릴 뿐이다 — 현행 유지.
- **JAMDC2 단일 실패점(SPOF)·강건성(원칙 6) 미모사 → [GitHub 이슈 #6](https://github.com/DaehyunY00/Air_Defense/issues/6)**: 현재 모델은 **융합허브(JAMDC2)가 마비·열화되는 상황을 전혀 모사하지 않는다.** KJADS 구축안 **원칙 6(강건성 및 복원력 — Degraded Mode, Fallback C2)**이 모델에 반영되어 있지 않아, To-Be는 허브가 항상 100% 가용하다고 가정한다. **감사 결과(Phase 4)**: 기본 스펙(c=6·8s·K=60)의 JAMDC2는 SC3 x3.0에서도 ρ≈0.49로 병목이 아니며(어떤 시나리오에서도 병목 아님), K는 전혀 구속하지 않는다(K를 20으로 낮춰도 동일).

  > ⚠️ **취약점으로 보고**: To-Be의 **누수율 개선폭은 융합허브 스펙에 상당히 민감**하다. 허브를 As-Is 수준 연산(c=3·서비스 25s)으로 낮추면 SC3 x2.5 To-Be 누수율 47.9%→72.7%로 As-Is(73.0%)에 거의 수렴하고, x3.0에서는 **역전**(74.5% vs 74.2%)한다 — 즉 누수율 우위의 상당 부분이 "허브를 후하게 준" 스펙에서 온다. 반면 **결심지연(204s→113s)·격추율 개선은 허브 스펙과 무관하게 견고**하다(자동화·사전승인·직결 라우팅 등 구조에서 옴). 논문 주장 시 "To-Be의 우수성"을 **누수율 단일 지표로 과대주장하지 말고**, 스펙 비의존적인 결심지연·구조 지표를 함께 제시해야 한다. 토네이도 민감도 순위(SC3 x2.5 누수율): 서비스시간(8→25s, 47.9→67.8%) > 서버(6→2, →63.8%) > 서버(6→3, →50.8%) ≈ 서비스(8→15s, →51.3%) ≫ 용량 K(60→20, 변화 0). `[C2-JAMDC2-SVC-01]`·`[ENV-DES-C2K-01]` 모두 등급 C. → SPOF·열화 모사는 별개 과제([이슈 #6](https://github.com/DaehyunY00/Air_Defense/issues/6)).

### 4-4. ⑥⑦ 결심·교전협조/권한위임(`feat/stage67-coord`)의 잔여 한계

⑥⑦을 [결심지연 분해 · coordPath 다익스트라 · 수평 교전협조 링크 · responsibility_gap 재설계 · 중복교전 모사]로 개편했으나, 이번 범위로도 **해결되지 않는 것**을 정직하게 기재한다:

- **기능적 교전통제권(ADAFCO형, KJADS 원칙 2-3) 미구현**: 협조 실패 시 "주교전자 1개 지정"은 **단순 규칙(최소부하·id 사전순)**일 뿐, 가중치 기반 우선순위 해소(어느 계통이 교전권을 갖는가를 무기 적합도·표적 근접·잔여 체공창으로 정하는 ADAFCO형 교전통제)는 없다. To-Be의 "Best-Shooter"는 무기 선택(⑧)에만 적용되고 계통 간 교전권 배분에는 적용되지 않는다. → 별개 과제.
- **AOR(교전책임구역) 시스템 미등록**: 지리적 교전책임구역이 데이터로 존재하지 않는다. 현재는 **축선(axis)이 대리 표현**하며, 중복교전 판정도 "동일 항적이 2계통에 팬아웃되었는가"로만 하고 실제 AOR 경계·중첩을 모사하지 않는다. 중복교전 위험의 공간적 정밀도는 축선 단위에 머문다.
- **교전의사 선언 메시지 미구조화**: `_coordCheck`는 협조 성립/실패를 **지연 임계(협조 지연 < 잔여 체공창)로만 추상화**한다. 실제 교전협조 프로토콜(교전의사 선언 → AOR 확인 → 주교전자 지정 → 상대 계통 사격중지)의 메시지 단계·왕복은 모사하지 않는다. 음성 협조의 왕복·오해·재확인은 단일 지연값에 뭉뚱그려진다.
- **중복교전 보존 모델의 단순화**: 중복교전은 "낭비된 병렬 사격"으로 모델 — 요격탄·engaged만 이중 계상하고 **격추/누수는 주 계통이 소유**한다(이중 격추 방지·보존 유지). 두 사격의 요격확률(Pk) 결합(둘 중 하나라도 맞으면 격추)은 모사하지 않는다. 따라서 중복교전의 비용(요격탄 낭비)은 계상되나, 중복이 격추율을 (미미하게) 높이는 효과는 반영되지 않는다 — 비용 측면을 보수적으로 본다.
- **책임공백 임계의 이중 정의**: 동적 DES(`_coordCheck`)는 "협조 지연 ≥ **잔여** 체공창", 정적 히트맵(`computeOverlapHeat`)은 "협조 지연 ≥ **0.5×dwell**"로 임계가 다르다. 둘을 통일하지 않고 나란히 유지해 상호검증한다(정적 예측 vs 동적 발생). 실측 상관은 강하나(위험 축선=발생 축선), SC1 seoul에서 동적 4건 vs 정적 0의 소폭 불일치가 있다(정의 차이의 엣지케이스).

### 4-5. ⑧ 교전/요격명령(WTA·명령링크·교전채널, `feat/stage8-wta`)의 잔여 한계

⑧을 [교전창 실현가능성 필터 · 축선(coverage)·사거리 필터 · 동점 통일 · engageTimeSec/channels/coverage 근거 문서화]로 개편했으나, 이번 범위로도 **해결되지 않는 것**을 정직하게 기재한다. **이 단계는 편향 방향이 엇갈려(As-Is를 좋게/To-Be를 나쁘게 하는 결함이 공존) Phase별 개별 기여도를 편향 원장으로 측정했다(`scripts/bias-ledger.mjs`).**

- **As-Is WTA의 전역 부하 조회(안 D 유지) — COP 부재 전제와 모순**: As-Is WTA는 전군 무기의 실시간 부하를 완벽 조회해 최소부하를 고른다. 이는 "각 군 자산이 실시간 통합되지 않는다"(KJADS GAP 5)는 As-Is 전제와 모순된다. Phase 3에서 대안 A(자기 서비스만 부하 가시)·B(서비스 우선 고정서열)·C(60초 낡은 부하)·D(현행)의 편향 원장 영향을 측정했다:

  | 안 | As-Is 격추 | To-Be 누수개선폭 Δ | 비고 |
  |---|---|---|---|
  | **D 현행(채택)** | 25.6% | **19.8%p** | 전역 실시간 부하 |
  | A ownchain | 15.6% | 27.4%p | COP 부재 최충실, 그러나 **개선폭 +7.6%p 확대** |
  | B priority | 23.3% | 21.4%p | 서열 근거 없음 |
  | C stale | 24.8% | 20.6%p | 정보 지연 표현 |

  안 A~C는 **모두 As-Is를 나쁘게 만들어 To-Be 개선폭을 키우는(결론에 유리한) 방향**이다. 근거 없이 채택하면 "고칠수록 결론이 좋아지는" 편향이므로, **근거가 가장 약한(GAP 5의 정량 모델이 없는) 현 시점에는 D를 유지하고 본 한계로 명시**한다. 동점 처리는 무조건 id 사전순으로 통일했다(3C — 종전 As-Is strict `<` 배열순 의존 제거).
- **`controlledBy[0]`만 사용 — 기능적 교전통제권(ADAFCO형, KJADS 원칙 2-3) 미구현**: 무기는 `controlledBy[mode][0]`(항상 첫 통제 C2)만 사용해, 하나의 무기를 복수 C2가 상황에 따라 통제하는 ADAFCO형 기능적 교전통제권을 표현할 수 없다. → ⑥⑦ 브랜치와 겹치는 별개 과제.
- **각도(angle)·교전기하 미반영**: KJADS 원칙 3-2의 "교전창·거리·각도" 중 교전창(Phase 1)·거리(Phase 2 축선)는 반영했으나 **교전 각도·기하(교차각·조우기하)는 미구현**. `wtaSuit`는 고도대역만 본다(Phase 4 미진행 — Phase 2 축선 필터로 무기 편중이 상당 부분 해소되어 재설계 보류).
- **유도탄 재고(magazine) 미모사 (Phase 5 미진행)**: `costPerShotM`은 누적만 되고 재고가 줄지 않아 무기가 무한 사격 가능하다. KJADS 원칙 5-1(비대칭 소진전술 대응)·5-2(임계치 관리)가 검증 불가. → 별개 과제(모델 범위 확대).
- **`engageTimeSec`의 단위 모호성**: FTR 300초는 근거 없는 스캐폴딩 값이며 **채널 점유시간이 아니라 스크램블 리드타임으로 의심**된다(`[WPN-*-ENGT-01]` 등급 C). Phase 1 교전창 필터가 이 모호성을 방어(300초여도 전체 격추율 견고)하나, 값 자체의 정합성은 미해결.
- **명령 링크 다중통제 미표현**: 명령 링크 지연은 ⑧ 카드에 아직 분리 표시되지 않는다(Phase 6 부수 수정 범위 — `commMeanDelay(res,'command')`·`maxWq(res,'shooter')` 추가 예정).
