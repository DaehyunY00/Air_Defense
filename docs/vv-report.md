# K-JAMDS C2 병목 분석 시뮬레이터 — V&V 보고서 (Phase 5)

> **디스클레이머**: 본 문서와 시뮬레이터의 모든 수치·좌표·확률은 공개자료(오픈소스) 기반의 **정책연구용 개념값**이며, 실제 작전자료가 아님. 모든 좌표는 도시 수준 개념좌표임. KP-SAM(신궁)·천마(K-31)는 탄도탄 요격 불가로 모델링하며, KAMDOC↔THAAD 연동은 모델링하지 않음. 본 시뮬레이터는 절대값 예측 도구가 아니라 **As-Is/To-Be 상대비교** 도구다.

## 1. Face Validation — 9단계 프로세스의 표준 프레임워크 정합성

계획서 Recommendations 1의 Phase 1 진입 게이트("9단계↔F2T2EA/OODA/TEWA 매핑표가 도메인 face validation을 통과")에 대응하는 매핑표. 구현 엔진(`js/engine/sim-engine.js`)의 각 단계가 국제표준 킬체인·의사결정 모델의 어느 요소에 대응하는지 명시한다.

| # | 구현 9단계 | F2T2EA (JP 3-01) | OODA | TEWA | 엔진 구현 지점 |
|---|---|---|---|---|---|
| 1 | 탐지 (Detect) | Find | Observe | — | `_beginDetect`/`_onDetect`: 축선·클래스 커버 센서, per-scan 탐지확률(detectFactor), 저탐지 위협 항적소실→재획득 반복 |
| 2 | 추적생성 (Track) | Fix·Track | Observe | — | `_onDetected`: 최속 보고경로(report 링크)로 담당 C2 전달, 링크 지연 반영 |
| 3 | 식별 (Identify) | Track | Orient | — | `_onC2Arrive`→C2 서버 처리 (M/M/c/K 서비스시간에 3·4·5단계 통합) |
| 4 | 위협평가 (TE) | Target | Orient | TE | 〃 (To-Be는 JAMDC2 융합 노드에서 AI 기반 식별·분류로 단축) |
| 5 | 무기-표적할당 (WTA) | Target | Orient/Decide | WA | `_doEngage`: canEngage 필터 + 최소부하 무기 선택 (TEWA의 WA 하위문제를 명시적 상태로 분리 — 계획서 Key Findings 1) |
| 6 | 결심 (Decision) | Target | Decide | — | `_decision`: 위협별 승인권자(approvalLevel) — As-Is는 상위 제대, To-Be는 위협별 자동화 차등(사전승인 자동교전 시 홉 생략) |
| 7 | **교전협조/권한위임** | (표준 6단계에 없음 — 본 연구의 핵심 보완) | Decide | — | `coordPath` BFS: As-Is 육↔공 음성 coord 홉(180초) — EADSIM의 SAM↔DCA deconfliction 별도 모델링 방식과 일치 |
| 8 | 교전/요격명령 (Engage) | Engage | Act | — | `_onShooterArrive`: 명령 링크 지연 + 무기 교전채널 M/M/c/K 처리 |
| 9 | BDA → 재교전 피드백 | Assess (→Find 폐루프) | Act→Observe | 폐루프 | `_onEngageEnd`: 요격확률(삼각분포) 판정, 실패 시 dwell 창 내 재교전(상한 3회) |

**정합성 판정**: 7단계 표준 모델을 유지하면서 (i) 결심과 교전명령 사이의 **협조/권한위임 서브단계**(한국군 이원화 C2 병목의 표현 수단), (ii) TEWA의 TE/WA 분리, (iii) BDA→재교전 폐루프를 추가한 9단계 확장이 그대로 구현되었다. 계획서 Key Findings 1의 권고와 일치.

## 3. Validation — 모델 타당성

### 3.1 극한값 검증 (계획서 4절 V&V)

- **위협 0 / 강도 0**: 생성·병목·부하 전부 0, NaN 없음.
- **초과포화 (강도 3.0)**: 이벤트 루프 정상 종료(무한루프 없음), 관측 ρ≤1(M/M/c/K 물리 상한), 드롭·누수 발생, 병목 도출 — 대기열 이론의 예측과 정성적 일치.

### 3.2 실증 사례 face validity — 2022.12.26 무인기 침투 재현 (SC1)

As-Is 모드에서 모델이 재현하는 구조가 공개된 사건 전개와 정성적으로 일치한다:

| 실제 사건 (공개자료) | 모델 관측 |
|---|---|
| 1군단 국지방공레이더 최초 포착, 항적이 국가방공체계에 미통합 | 육군 센서(LLR-1C) 탐지 → AOC-1C 보유 항적이 MCRC로 음성(180초) 협조로만 전달 — 해당 voice 링크가 통신병목으로 도출 |
| 공작사·항작사·군단·수방사 개별 작전(중복·책임공백) | 중복교전 히트맵: 서부·수도권 축선에서 As-Is 위험>0, To-Be(JAMDC2 융합)에서 0으로 해소 |
| 격추 실패 (헬기·전투기 약 20대 투입) | 소형 무인기 요격확률 Triangular(0.1,0.3,0.5) — 표본 실행에서 무인기 격추율이 타 위협 대비 현저히 낮음 |
| 5시간 이상 체류 | dwellSec=900(최장) + 저탐지(detectFactor 0.4) 재획득 반복 |

### 3.3 민감도 분석 (±20% 스윕, 계획서 4절)

SC3 포화 시나리오(As-Is, 강도 1.5)의 누수율 스윙: **노드 처리시간 ~28%p ≫ 위협 강도 ~18%p ≫ 요격확률 ~9%p ≫ 통신지연 ~1.5%p ≈ 탐지확률 ~0.6%p**.
- 포화 구간의 지배 인자는 처리용량 — "병목은 처리용량"이라는 진단의 정량적 근거.
- 대조: 탐지 제약 시나리오(SC1)에서는 탐지확률↑→누수↓ 단조성이 성립 — 민감도 구조 자체가 시나리오의 함수임을 확인(모델이 사전 결론을 내장하지 않음).

### 3.4 임계 전환점 — As-Is→To-Be 효과 정량화 (계획서 Recommendations 6, 핵심 산출물)

SC3 복합 섞어쏘기 포화, 강도 0.5×~3.0× 스윕(11점×2모드×30복제, seed=12345, 결정론적):

| 구간 | As-Is C2 최대 ρ | As-Is 누수율 | To-Be 누수율 | 개선폭(gap) |
|---|---|---|---|---|
| ×0.5 (저강도) | 0.33 | 44.4% | 36.4% | 8.0%p |
| ×1.5 (임계 직전) | 0.86 | 54.8% | 41.6% | 13.2%p |
| **×1.75 (ρ≥0.9 돌파)** | **0.93** | 60.4% | 42.6% | 17.8%p |
| ×2.5 (최대 격차) | 0.98 | 70.1% | 49.3% | **20.8%p** |
| ×3.0 | 0.98 | 72.6% | 56.2% | 16.4%p |

- **임계 이전 평균 개선폭 10.3%p → 임계 이후 18.9%p** (약 1.8배).
- 정책 함의: To-Be(Track Fusion·자동 WTA·데이터링크 30초→2초)의 가치는 평시·저강도가 아니라 **결심권자 이용률이 임계(ρ=0.9)를 넘는 포화 구간에서 비선형적으로 발현**된다. 이것이 K-JAMDS 투자정당화의 정량 논거다.
- 대조군: SC5 저강도 기준선은 전 스윕 구간 ρ<0.9(전환점 부재) — 전환점 역시 시나리오에서 도출되는 값이지 고정 속성이 아니다.
- UI: [Monte Carlo] 탭 "임계 전환점" 카드에서 대화형 재현 가능(동일 seed → 동일 곡선).

## 4. 한계 (계획서 Caveats 승계 + 구현 고유)

1. **개념값 기반**: 파라미터 다수가 신뢰도 C(개념 추정)·B(2차 보도) — 절대값이 아닌 As-Is/To-Be 상대비교로만 해석할 것. 신뢰도 등급은 `docs/params.md` 전 항목에 명시.
2. **모델 단순화**: 3·4·5단계(식별·위협평가·WTA)를 단일 C2 서비스시간으로 통합, 지수 서비스시간 가정(M/M/c), 교전은 요격확률 단발 판정(비행시간·기하 미모델링), 위협 궤적은 시각화용 선형보간.
3. **합참지 제104호 원문 미확보**: K-JAMDS 세부 약어(JAMDC2·KIECS 등)는 사용자 제공 개념으로 처리(계획서 Caveats).
4. **trace 절삭**: 재생용 위협 추적은 300건, 노드 시계열은 20,000샘플 상한(초과 시 truncated 플래그 명시) — **통계 결과는 절삭과 무관하게 전체 모집단 기준**(회귀로 검증).
5. **Mulberry32**: 32비트 주기 — 본 규모(복제 수백 회)에는 충분하나 초대규모 배치는 장주기 생성기 검토 필요.
