# ADR-051: FULL As-Is 군단 AOC 방공 C2A 정보상태

- 상태: Accepted
- 일자: 2026-07-21
- 범위: 고해상도 `highResolutionDeployment=true` As-Is

## 문제

기존 FULL As-Is는 MCRC와 `ARMY_LOCAL_AD`가 동일 위협을 독립 처리하는 결과는 내지만,
군단 AOC의 `MCRC 항적+국지레이더 항적` 융합과 `AOC 교전현황→MCRC` 제한형
공유 메시지가 없었다. 중복교전은 정보 지연·드롭이 아니라 단순 복수 계획 병존으로만 발생했다.

## 결정

1. FULL의 5개 `ARMY_LOCAL_AD`를 수량 변경 없이 `CORPS_AOC_C2A` 아키텍처 역할로 명시한다.
2. MCRC→각 AOC에 As-Is 16초 개념 항적 `report` 경로를 추가한다.
3. AOC는 최속 국지 출처 1개와 MCRC 유래 출처 1개를 동일 항적으로 상관·융합한다.
4. 융합항적은 위협군과 잔여 방어시간으로 우선순위를 부여하고 C2 대기열에서 고우선순위를 먼저 처리한다.
5. AOC는 자신의 권역·능력·PIP·FC·탄약·채널을 통과한 자산만 자동 할당·승인한다.
6. AOC→MCRC 교전현황은 As-Is 음성/VTC 채널로 전파한다. 채널당 서버 1, 처리 중 포함 수용 4건, 지연 삼각분포 90/180/270초, 신선도 300초다.
7. MCRC는 수신한 AOC `assigned/fired` 상태가 신선하면 사수선정 또는 발사 직전에 중복을 해소한다. 전파 지연·드롭·신선도 초과로 상태를 모르면 독립 교전한다.
8. MCRC 무력화 배치에서는 MCRC 유래 융합과 MCRC 현황공유 링크를 모두 제거한다.

## 관측 지표

- `coordination.trackFusion`: 보고 수신, 융합항적, 복수출처 융합, 우선순위 처리
- `coordination.statusSharing`: 전송, 수신, 대기, 드롭, 수신 시 신선도 초과, 중복 해소, 상태 불일치 기인 실제 중복교전

## 호환성·한계

- legacy OFF 결과 형상·RNG·Phase 0 해시는 변경하지 않는다.
- 군단별 실제 AOR·교전규칙·메시지 포맷이 아닌 권역·축선 기반 공개자료 개념 모델이다.
- 항적 상관은 최속 국지 1개+MCRC 1개의 결정론적 결합이며, 오상관·항적 분리 확률은 후속 범위다.
