# ADR-052 — Legacy ICC–ECS–MFR–포대 10세트 확장

## 상태

채택 — 2026-07-21

## 결정

기본 `legacy` 배치에 ICC, ECS, 전속 MFR, 천마/천궁-II 포대를 한 세트로 묶어 10세트를
추가한다. “약 2개씩·총 10개”는 서부 4·중부 3·동부 3으로 해석하고, 천마 5·천궁-II
5를 권역별로 교차 배치한다. 기존 24노드에 40노드가 추가되어 총 64노드가 된다.

각 세트의 ID는 `ICC-<권역키>`, `ECS-<권역키>`, `MFR-<권역키>`,
`BAT-<체계>-<권역키>`로 고정한다. As-Is는 MFR→ECS→ICC와 ICC↔MCRC 180초
음성/VTC 협조를, To-Be는 MFR·ICC→JAMDC2 2초 데이터링크와 통합항적을 사용한다.

## 근거와 경계

- 좌표는 도시·권역 수준 개념점이며 실제 군사시설 좌표나 실제 전력 수량이 아니다.
- 기존 천마·천궁-II의 능력, Pk, 탄약, 채널 파라미터를 재사용한다.
- 천마의 탄도탄 교전 불가 제약은 유지한다.
- `MRF`라는 요청 표현은 코드·문서의 표준 약어 `MFR`로 통일한다.
- 신규 MFR을 별도 센서로 실행하므로 As-Is의 중복항적·음성협조 부하는 의도적으로 증가한다.
  이를 제거하는 것은 배치 추가가 아니라 Track Fusion 변경이므로 To-Be에서만 해소한다.

## 결과

- 중앙축 UAV의 FTR 단독 취약은 천마 3개, 동부축 대구경방사포의 MDU-L 단독 취약은
  천궁-II 2개가 보강한다.
- 저강도 처리 노드는 포화되지 않지만 ICC→MCRC 180초 링크는 통신병목으로 관측될 수 있다.
- 저배율 지도에서는 같은 사이트의 네 마커가 중첩되므로 데이터 좌표는 유지한 채 표시점만
  방사형으로 분리하고 실제 좌표에 리더선을 연결한다. zoom 11 이상에서는 실제 좌표로 복귀한다.
- 지도 범례에 활성 노드·10세트 포함 여부를 표시하고, 연결선은 과밀 방지를 위해 기본 OFF로 둔다.
  SVG fallback에도 동일한 표시 분리와 탐지·교전 범위 링을 제공한다.
- 기존 legacy 결과 해시는 의도적으로 폐기하고 `tests/phase0-baseline.json` schemaVersion 2와
  `tests/legacy-snapshot.json`으로 재기준선화한다. 플래그 생략과 명시적 고해상도 OFF의
  bit-exact 동등성은 계속 보장한다.
