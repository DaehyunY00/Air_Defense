# ADR-004 — timeout 분해 + overflow:shooter 재분류 (Phase 4, ⑨)

## 맥락
누수 사유 `timeout`(처리지연 초과·공역이탈)이 **두 가지 물리적으로 다른 현상**을 한 코드로 뭉뚱그린다(사실 e).
- **교전을 한 번도 못 함**(`tries===0`): 앞단 ②~⑦ C2·협조·승인이 시간을 소진 → **구조적**(파이프라인 결함).
- **교전했으나 명중/체공창 소진**(`tries>0`): ⑧⑨ 교전·BDA 단계의 물리 한계 → **비구조**(무기 성능·체공 시간 문제, C2 무관).

또한 `overflow:<노드>`(대기실 초과 손실)를 taxonomy가 **일괄 구조적**으로 분류하는데, 노드가 **교전채널(shooter)**이면 이는 C2 처리 포화가 아니라 유도탄·발사대 수 한계(=`no_shooter` 계열) → **비구조**로 봐야 한다.

## 선택지
- **A. tries 기준 분해**: `timeout:c2`(tries===0, 구조) / `timeout:engage`(tries>0, 비구조). overflow는 노드 category로 재분류.
- **B. 분해하지 않음**: 단일 timeout 유지 — 구조/비구조 혼재를 감수(정보 손실).
- **C. 시간·단계 계측 기반 분해**: 각 위협이 어느 단계에서 시간을 얼마나 썼는지 적분 → 지배 단계로 귀속. 구현 복잡, 이벤트 계측 추가(동역학 무관하나 상태 증가).

## 결정
**A** — `_onExit`에서 `timeoutSplit`(기본 ON) 플래그로 timeout을 `tries` 기준 분해. `overflow:shooter`는 `leakTaxonomy`에서 노드 category로 재분류(비구조). taxonomy에 `timeout:c2`(구조, 단계 '②~⑦ 파이프라인')·`timeout:engage`(비구조, 단계 '⑧⑨ 교전·BDA') 추가.

**되돌리기 경계(중요):**
- **동역학 지문(sp/k/l/iM/ex) — 완전 불변, 0/36.** 두 변경 모두 순수 보고/분류 변경이다.
- **timeout 코드 방출** — `timeoutSplit=OFF` → 단일 `timeout`(구조) 방출 = legacy. 완전 되돌림.
- **overflow:shooter 재분류** — `leakTaxonomy`는 UI·테스트·원장이 공유하는 **단일 분류원**이라 플래그로 게이트하지 않고 **무조건 교정** 적용. 근거는 아래 “근거”의 마지막 항.

## 근거
- **분해 기준은 ⑧ `no_engage_window`와 정확히 동일**(`threat.tries > 0` → 비구조). 두 브랜치가 같은 판정을 쓰는지 확인 완료 — `timeout:engage`와 `no_engage_window` 모두 `structural:false`이며, 어서션으로 일관성을 고정(`tests/reengage.test.js`).
- overflow:shooter 재분류를 **게이트하지 않은 이유**: `leakTaxonomy`는 정적·무상태 분류 함수로 UI(대조표·타임라인·⑨카드)·회귀 테스트·편향 원장이 **단일 소스**로 공유한다. 플래그를 주입하려면 7개 호출부에 분기를 심어 taxonomy를 포크해야 하고, 이는 “정본 분류” 원칙을 깬다. 재분류는 **방출 코드를 바꾸지 않고**(overflow:MDU-L은 legacy에서도 방출됨) **라벨·구조성 해석만** 교정하며, **동역학·지문에 영향이 없다**(0/36). 근거 등급 **B**(무기 교전채널 = 유도탄·발사대 수, 문서 [WPN-*-CHAN-01]). 결론 이동은 아래대로 **+2.5p**(전체 +7.0%의 일부)로 20% 미달 → 조건 2의 “C등급 & 유의미 이동 → 기본 OFF” 대상이 아니다.

## 결론 영향 — ⚠️ 과제 예측과 불일치(정직 보고)
과제는 Phase 4가 “🔴 에스컬레이션 임계를 확실히 넘는다”(To-Be 구조적 565→9, ~97% 급감)고 예측했다. **실측은 그렇지 않다.**

편향 원장(seed1~20 × SC1/2/3 × x1.0/1.5/2.5 풀링, `scripts/phase4-measure.mjs`):

| 규칙 | As-Is 구조적실패율 | To-Be 구조적실패율 | To-Be 개선폭(As-Is−To-Be) |
|---|---|---|---|
| legacy(Phase 3) | 44.25% | 19.16% | 25.09p |
| +timeout 분해 | 42.47% | 17.24% | 25.22p |
| **+overflow 재분류(Phase 4)** | **41.59%** | **14.74%** | **26.85p** |

- **To-Be 개선폭 이동: 25.09p → 26.85p = 상대 +7.0% → 20% 미만, 🔴 아님.**
- **왜 예측(−97%)이 빗나갔나:** 과제의 사실 (e)(“To-Be timeout의 98%가 `tries>0`”)는 **⑧ 병합 이전** 측정값이다. ⑧ 교전창 필터(`feat/stage8-wta`, 이미 main 병합)가 체공창 내 교전 불가 위협을 **교전 개시 전**에 `no_engage_window`(tries===0)로 우회시킨다. 그 결과 To-Be의 남은 timeout은 대부분 `tries===0`(=`timeout:c2`, **구조적 유지**)이고, 비구조로 재분류되는 `timeout:engage`(tries>0)는 소수다. 즉 **⑧이 “교전창 부족”을 이미 별도 코드로 떼어내, Phase 4가 떼어낼 몫을 선점**했다.
- 표본(SC3 x1.5, seed1~5): timeout 분해 346 = c2(구조) 311 + engage(비구조) 35 — 합 보존. 구조적 감소는 346→311(10%)에 그친다.

**결론: Phase 4는 에스컬레이션 임계를 넘지 않는다(+7.0%). 과제의 🔴 예측은 ⑧ 병합으로 무효화됐다.** 이는 “판단은 위임하되 추적 가능성은 위임하지 않는다” 원칙에 따라, 예측을 강행하지 않고 실측대로 보고한 결과다.

## 되돌리는 법
- 플래그: `features.timeoutSplit=false` → 단일 `timeout`(구조) 방출 = legacy 코드 분포. 동역학 지문 0/36 유지.
- overflow:shooter 재분류는 무조건 적용(단일 분류원)이나 방출 코드·동역학·지문 불변 → 회귀 안전. 원복하려면 `leakTaxonomy`의 `isShooter` 분기를 제거(1줄).
